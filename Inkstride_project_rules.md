# Inkstride プロジェクト 引き継ぎドキュメント

## このプロジェクトの目的

**枯れた普遍的な技術で、 自分の環境と好みの UI を実現する 2D ペイントツール**を作る。

- iPad + Apple Pencil で動く WebGL2 ベースの単一 HTML ファイル ペイントアプリ
- 個人利用が目的
- **先端技術や独自アルゴリズムは求めていない**
- 既存の枯れた技術 (drawImage、 アフィン変換、 ホモグラフィ等) で十分

---

## 絶対ルール (作業時に必ず守ること)

### 1. トークン節約を最優先

- 不要な前置き・解説・自慢話を書かない
- 結論を先に、 詳細は最小限
- 同じ説明を何度も繰り返さない
- 確認のための再 view は最小回数に絞る

### 2. チャットが途切れても引き継げるようにする

- 重要な決定事項は **このドキュメントに追記**して保存
- バージョンごとの変更内容は **コード内コメント**に明記 (`// vXX: 〜`)
- 巨大な変更前には必ず**現状の重要ポイント**をメモに残す

### 3. メモリが途切れたら勝手に深追いしない

- 文脈が不明な場合は **ユーザに報告して選択肢を提示**
- 「これは v○○ で対処済みでしたか？」 と確認
- **前提が崩れたら、 推測で進めずに止まる**

### 4. 暴走して試行錯誤を繰り返さない

- **同じ問題に 3 回以上同じアプローチで失敗したら、 必ず立ち止まる**
- ユーザに状況を報告し、 **アプローチを変えるかどうか確認**
- 「やってみないとわからない」 という言い訳で連続実装しない

### 5. 選択肢がある場合は必ず明示

- 実装方針が複数考えられる場合、 **A 案 / B 案 / C 案** を列挙
- 各案のメリット・デメリットを 1〜2 行で
- ユーザの選択を待ってから実装

### 6. トークンを大量消費する作業は事前報告

- ウェブ検索を多数行う必要がある場合 → **事前に報告して許可を得る**
- ドキュメント参照が必要な場合 → **何を調べたいか明記**
- 大規模な書き換えが必要な場合 → **見積もりを提示**

### 7. 過剰な機能追加・最適化を避ける

- ユーザが要求していない機能を**勝手に追加しない**
- 「ついでに」 と称して別の修正を入れない
- 「より高度な」 アプローチを提案しない

## 過去の失敗パターン (繰り返さないこと)

### A. 強制フェードでインクだまりを解決しようとした (v40〜v49)

- `_strokeStampCount` で始点スタンプの opac を強制的に下げる修正を繰り返した
- ユーザが「不透明度感知オフでシャープな線が引きたい」と言っているのに、 強制的に opac を下げてしまった
- **教訓**: ユーザの設定意図を曲げる修正は絶対にしない

### B. インクだまり問題を「アルゴリズムで完全解決」 しようとした

- 物理筆圧の特性で確率的に出るものを、 補正で完全に消そうとして失敗
- **正しい解**: 業界標準どおり、 距離ベースの入り抜き機能 (v51) を提供
- **教訓**: 業界標準アプリ (Procreate/CSP/Krita) が**やっていないこと**は、 やっても無駄

### C. ボタン位置調整を 10 回以上繰り返したのに被りが直らなかった (v54〜v58)

- 「ボタンが被ってる」 と言われても、 真の原因 (テキスト幅と CSS width の不整合) に気付かず位置だけずらしていた
- **教訓**: 「同じ問題が 3 回直らなかったら別アプローチ」 を厳守

### D. 「ついでに」 高速化したら品質低下 (v52〜v54)

- 滑らかさ向上のためバイキュービック補間 + 32x32 サブディビジョンを実装
- 結果 5 秒/コマの激重に → ホモグラフィ直接計算に変更してようやく解決
- **教訓**: 単純な手法 (アフィン変換+8x8 程度) で十分なことが多い

### E. 不要な機能を勝手に追加

- ワープ変形に「辺中点で曲線」 を追加 → ユーザは直線のパース補正だけ求めていた
- **教訓**: ユーザの仕様を勝手に拡張しない

## 設計原則

### アルゴリズム選択

| 用途 | 採用 | 不採用 |
|---|---|---|
| 描画 | アフィン変換 + drawImage | WebGL シェーダ自前実装 |
| 補間 | 中点ベジェ・直線 | Catmull-Rom・バイキュービック (重い場合) |
| 変形 | ホモグラフィ (4頂点) | 5x5 メッシュバイキュービック (重い) |
| 入り抜き | 距離ベース線形フェード | 強制スタンプフェード |

### UI 設計

- **ボタンラベルは短いアルファベット** (SIZE/FIT/PLAY 等)
- **ピクセル数等の長文は設定パネル内**に
- ボタンサイズ: 60×26px、 6px 隙間で**個別配置**
- ボタン間の隙間ではキャンバス操作可能

## 現在の状態 (v58 時点)

### 完成している機能

- ペン/マーカー/消しゴム描画 (中点ベジェ補間)
- 距離ベースの入り抜き機能 (TOOLS.startTaper / endTaper)
- 始点無視デッドゾーン (TOOLS.deadZone)
- ホモグラフィ・ベースのワープ変形 (4 頂点 + 4 辺平行移動)
- 5x5 メッシュ・グリッド変形 (バイキュービック描画)
- レイヤー、 アンドゥ、 ギャラリー保存、 PNG/PSD 出力
- リファレンス画像、 タイムラプス再生

### 未解決の課題

- **インクだまり**: 物理筆圧の特性で確率的に出る。 距離ベース入り抜きで対症対応済み。 真の解決は不可能と判断
- **上部ボタンの被り**: v58 時点で未確定。 ユーザが他の作業に切り替え

### コード構造

- 単一 HTML ファイル `Inkstride_v21_NN.html` (約 6300 行)
- 大部分は単一 `<script>` ブロック内の JavaScript
- WebGL2 + Canvas2D ハイブリッド
- localStorage で設定永続化

## ユーザの方針 (重要)

- iPad と Apple Pencil で個人的に使うアプリ
- アプリ開発自体が創作モチベーションの回復になっている
- **シャープな線が引きたい** (強制フェードは絶対 NG)
- **シビアな実用条件で安定して動く**ことが最重要
- 派手な機能より、 **基本動作の信頼性**
