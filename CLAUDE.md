Project rule summary:

- Keep changes small and practical.
- Do not add unrequested features.
- Do not weaken sharp strokes with forced fade or opacity compensation.
- Present options before large or ambiguous changes.
- Stop after 3 failed attempts on the same issue and ask the user.
- Record important decisions in docs and add short version comments near code changes.
